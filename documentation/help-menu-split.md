# Split Help Menu into User and Admin Sections

## Summary

The help menu is currently too large and mixes user and admin commands.

## Desired Behaviour

-   `/oma help` shows core commands
-   `/oma help admin` shows advanced/debug commands

## Tasks

-   Create separate help functions
-   Update slash command handling
-   Add hint for admin help

## Acceptance Criteria

-   Default help is concise
-   Admin help accessible via command
